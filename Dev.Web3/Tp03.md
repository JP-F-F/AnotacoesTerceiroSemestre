# TP03 - Versionamento Gihtub, Git, Gitlab

Um **Sistema de Controle de Versão** (também conhecido como Sistema de Controle de Versionamento), do inglês VCS – Version Control System ou SCM – Source Code Management é um software que gerencia versões de arquivos.

Geralmente empresas que escolhem VCSs comerciais ao invés dos open source é que eles oferecem uma garantia, enquanto os open source não se responsabilizam por erros/perdas de informação.
De qualquer modo atualmente as soluções open source estão tendo um desempenho e segurança maior.

## Tipos de software de versionamento

Os softwares de versionamento são classificados em gerações.
* 1º Geração: rastreavam alterações em arquivos de forma individual, a edição era feita de forma local e por um usuário por vez. O proposito de sua criação era que cada usuário efetuasse o login em um nó Unix comum.
* 2º Geração: apresentavam suporte de rede, permitindo a criação dos repositórios centralizados com versões “oficiais”, permitindo que múltiplos usuários desenvolvessem ao mesmo tempo o projeto o que exigia uma conexão com a rede constantemente.
* 3º Geração: nesta geração os repositórios são do **tipo distribuídos**. Todas as copias de repositórios são consideradas iguais, não há um repositório central. O que permite abrir caminho para confirmações ramificação e mesclagens criadas localmente sem acesso à rede e movidas para outros repositórios centralizados (de acordo com a necessidade).

**Sistema Local de Controle de Versão**: Consiste em copiar a versão em um diretório local no computador, com um arquivo de texto contendo informações relevantes como data, hora, envolvidos e o que modificou.

**Sistema Centralizado de Controle de Versão**: Consta como um único repositório central, interligando os desenvolvedores através de uma topologia estrela. As copias do desenvolvimento são alocadas de forma local e comunicam com o repositório central.
**Commit:** Quando se envia uma versão para o repositório central;
**Update:** Quando se retira uma versão do repositório central;

Dentre as inúmeras opções de sistema de versionamento centralizado, os dois mais utilizados são os CVS e SVN no qual merece uma boa atenção para quem desenvolve soluções em equipe e centralizada.

## Git

Pela _documentação oficial_, Git é uma ferramenta de versionamento de código distribuído de código aberto e gratuito, projetado para lidar com tudo, de projetos pequenos a grandes.
O Git é um padrão de mercado e surgiu pelo desgosto do Linus Torvalds com os softwares de versionamento da época.

Hoje existe o _Git e o GitHub_ (porém ambos são **DIFERENTES**, não confundam). Tais tecnologias são normalmente apresentadas juntas em cursos, etc. porém **não são a mesma coisa**, são tecnologias completamente diferentes.

A principal funcionalidade do Git, que o faz ser amplamente utilizado em projetos de desenvolvimento de software, é a possibilidade de fazer o **controle de versões de modo colaborativo**, ou seja, é possível que o mesmo arquivo seja modificado ao mesmo tempo por dois desenvolvedores diferentes, e que ambas as alterações sejam salvas **sem que nenhum código seja sobrescrito.**

Para permitir o modo colaborativo, o Git utiliza o conceito de **ramificação ou branch**, onde cada branch é uma linha do tempo que possui marcos ou commits, e nesse branch os arquivos podem ser alterados livremente sem impactar outras ramificações.
